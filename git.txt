git (1)
概念
特点

git 是一种文件版本控制系统。
最大的特点是分布式。在本地支持完整的数据库，不依赖于服务器，给了用户最大的自由。可以随时随地的保存自己的工作。
实现的很轻量级，但功能十分强大，管理了世界上最大的开源项目linux kernel.

还有很强大的github.com托管代码，真正的做到了，可以在世界上任何有网络的地方工作。十分酷的事情。

1 本地文件版本控制
安装好git后，就可以用它来进行版本控制，或者跟踪，记录文件状态了。git是依托文件系统的。不用把它想成很复杂的系统。想成一个记录你文件变换的小工具就可以了。

首先在本地设置一个目录作为工作或者玩耍的地方。只需要一个命令就可以在这个目录上建立跟踪设施 (>git init). 然后随心所欲的在这个目录里折腾就好了，不要删除里面的隐藏的.git目录就好。当折腾累了想保存工作，等过一阵子再玩的时候你就可以用这个工具来做一次跟踪，也很简单(>git add *.*). 给这个目录下所有的文件做跟踪。然后(>git commit), 增加一个comment就完成了工作了。以后想看今天的工作，可以很快的回到今天的状态。

当然为了方便工作，还提供了一些辅助的工具，比如(>git log),(>git status), 来查看所做过的commit,或者现在跟踪的状态。
